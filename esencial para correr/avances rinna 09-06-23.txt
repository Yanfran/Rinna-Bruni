NOTA: El error que estás viendo se debe a que la fecha '1922-09-19' Se considera una fecha antigua y MySQL tiene limitaciones en cuanto al rango de fechas 		(QA-WEB-86 - Ingreso de fecha incorrecta - bug)
	SE DEBE AGREGAR UNOS COMANDO 



El error que estás viendo se debe a que la fecha '1922-09-19' se considera una fecha antigua y MySQL tiene limitaciones en cuanto al rango de fechas que 		puede manejar por defecto. El error que estás viendo es específico de MySQL y se debe a que la fecha que estás intentando insertar está fuera del 		rango de fechas admitido por MySQL por defecto.

	Para solucionar este problema, puedes considerar las siguientes opciones:

	Actualizar la configuración de MySQL: Puedes modificar la configuración de MySQL para permitir fechas fuera del rango predeterminado. Para 		hacerlo, debes cambiar el modo de SQL tradicional a modo ANSI. Agrega la siguiente línea en la configuración de MySQL (generalmente en el archivo 		my.cnf o my.ini):

	makefile
	Copy code
	sql_mode = "NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION"
	Luego, reinicia el servidor MySQL para que los cambios surtan efecto.

	Usar un formato de fecha diferente: En lugar de usar el formato 'YYYY-MM-DD', puedes usar otro formato de fecha, como 'YYYYMMDD' (sin guiones ni 		separadores) al insertar fechas antiguas. Luego, cuando necesites mostrar o manipular las fechas en tu aplicación, puedes formatearlas 		adecuadamente.

	Almacenar la fecha como texto: Si no necesitas realizar cálculos basados en estas fechas y solo las estás almacenando para fines de visualización, 	puedes considerar almacenar la fecha como una cadena de texto en lugar de un valor de fecha. Sin embargo, esto no es recomendable si planeas 	realizar operaciones de fecha en la base de datos.

	La opción que elijas dependerá de tus necesidades y restricciones específicas del proyecto. Asegúrate de comprender las implicaciones de cada 	opción antes de implementarla.
